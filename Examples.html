Base Application Behavior


The file Base-Application.maude contains the modulo RULES whos define the dinamic behavior of distributed reactive base appilcation, we can do the following to enter it: 
 
maude> load Base-Application.maude
Affter entering the module RULES into Maude, we can, for example define a simple everioment: there are two Producer-Observables and three Consumer-Observables that issue data flow to the system. 
 
 < 0 : Producer | cnt-prod: 0,frequency: 1,clock-p: (0 0 0 0 0),weights: 5,end: 4,finish: false,Notice-logs: nil >
 < 1 : Producer | cnt-prod: 0,frequency: 3,clock-p: (0 0 0 0 0),weights: 1,end: 3,finish: false,Notice-logs: nil >
 < 2 : Consumer | current-mesg: none,freq-C: 2,actual-time: 0,logs: nil,limit: 34, clock-c:(0 0 0 0 0) >
 < 3 : Consumer | current-mesg: none,freq-C: 1,actual-time: 0,logs: nil,limit: 30, clock-c:(0 0 0 0 0) >
 < 4 : Consumer | current-mesg: none,freq-C: 5,actual-time: 0,logs: nil,limit: 55, clock-c:(0 0 0 0 0) >
This example is commented on at the end of the file, you can eliminate the --- then when you load the file Maude is going to use rew comand to explore the behavior of tha above example, keep in mind that each different execution may result in a different result. The result present information about the total rewrites that Maude did and the result of that posible trace i.e a application of consecutive rewiting rules.

This rewrite example shows a possible execution for the system pesented above, as you can see the producer fished its execution and the atributte Notice-logs of each producer gives information about that and the Consumers consunme diferents amount of messages due to its different initial configurations. Note that at this point we only load the base behavior of the reactive base application witout observers, the system only issue data flows bettewn the diferents observables. If you want to test a particular configuration you can change the configuration of the initial state of the system and reload the module.

Observer Behavior

The file Observers.maude contains the modulo OBSERVER whos define the dinamic behavior of distributed reactive observer over a base application, this module automatically loads the modules Base-Application.maude , Temporal-logic.maude , Predicates.maude , we can do the following to enter it: 

 
maude> Observers.maude
