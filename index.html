DRRF


We study the semantics of distributed reactive programming languages using rewriting logic. We propose a reactive event-based programming framework with explicit support for distribution, concurrency, and explicit time manipulation DRRF which was developed on Maude. The framework uses rewriting logic and includes a model of the distributed base application, a model of observables and observers, and a model for complex predicate constructs supporting explicit time manipulation.


Prerequisites


The Maude system is free software, is a high-performance reflective language and system supporting both equational and rewriting logic specification and programming for a wide range of applications. Maude Systemf runs on linux and Mac OS, is avaliable at the Maude home Page.


A maude session can be started by calling the maude.linux64 binary in the Maude dictionary from the Linux shell window, the procces is similar for other plataforms. For example we can move to the dictionary and invoke Maude and we can see the version of the system being executed, the date it was built, corpytight information, and the current date.



            ~/Maude-2.7.1-linux$ ./maude.linux64
                              \||||||||||||||||||/
                            --- Welcome to Maude ---
                              /||||||||||||||||||\
                    Maude 2.7.1 built: Jun 27 2016 16:35:16
                      Copyright 1997-2016 SRI International
                            Tue Mar 12 11:30:31 2019
            Maude>

Then the Maude Sytem is ready to accept Maude modules and commands. During the session, the user interacts with the system by entering a request at the Maude prompt.

Modules


DRRF framework is composed of several Maude modules that model the dynamic behavior of a distributed reactive system.


  • Base-Application.maude: specify the behavior of an environment where the observables issue data flow toward the system.
  • Observers.maude: formal model for a distributed observer that watches the data flow issue by an observable.
  • Temporal-logic.maude: formal model for LTL formulas.
  • Predicates.maude: specify LTL predicates.
  • Vector.maude: specify vector clocks using in the system.
